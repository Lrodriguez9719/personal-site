This is a weekend project I built in order to prepare for a technical interview I had for a NodeJS development role. I didn't get the job, but I did learn a lot in the process haha.

The main focus of the interview was about what I would do if I had to build a video streaming endpoint with NodeJS. They gave me a few days in order to prepare and because I like to always do my best, I decided to build both the backend for the video streaming endpoint, and a small frontend in order to consume said endpoint and in consequence build a "complete" video streaming web application.

What's funny (at least for me) is that during the interview I didn't show them this project. Although when they did asked me how I would implement such an endpoint, I think I did a pretty good job at explaining the main concepts and steps on how to build it, that's what I take away from building this project. I was able to learn about the HTTP 206 Partial Content status code, how to use Node.js Streams to efficiently stream video files, and how to handle range requests from clients.

As someone who almost always has worked on the frontend side of web applications, this project was a lot of fun because it allowed me to do a dive little bit deeper into backend development (and DevOps a bit as well since I had to deploy the project to a VPS). Overall, I think it was a great learning experience and I'm glad I took the time to build it.

Now onto learn more new things and build more projects!
